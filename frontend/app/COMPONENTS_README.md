# Frontend Refactoring: Component Modularization

This document outlines the changes made to the `frontend` folder to improve code maintainability, readability, and reusability by separating the monolithic `page.tsx` into smaller, focused components.

## Overview of Changes

1.  **Monolith to Modular:** The main dashboard in `app/page.tsx` was broken down into five distinct UI components.
2.  **State Management:** `page.tsx` now acts as a **Smart Container**, managing the business logic (fetching data, CRUD operations, state), while passing data down to **Presentational Components** via props.
3.  **Style Extraction:** Global UI styles (like the custom scrollbar) were moved from inline `<style>` tags to `globals.css`.

## New Component Architecture

The following components were extracted/updated in `app/components/`:

### 1. `Header.tsx`
- **Purpose:** Displays the application logo, "FINASTRIVA" brand name, and the current dashboard date.
- **Key Features:** Hover effects on the logo and responsive date display.

### 2. `BalanceCard.tsx`
- **Purpose:** Shows the "Total Balance", "Income", and "Expense" summaries.
- **Logic:** Dynamically changes text color to red if the total balance is negative.

### 3. `AnalyticsCard.tsx`
- **Purpose:** Visualizes the allocation of funds using a Donut/Pie Chart.
- **Dependencies:** Uses `recharts` for responsive data visualization.
- **Feature:** Handles "Empty State" gracefully when no transaction data is available.

### 4. `TransactionForm.tsx`
- **Purpose:** A dual-purpose form for adding new transactions or editing existing ones.
- **Props:** Receives form state (`desc`, `amount`, `type`) and handler functions from the parent.
- **UI:** Includes toggle buttons for switching between "Income" and "Expense".

### 5. `TransactionList.tsx`
- **Purpose:** Displays a scrollable history of recent transactions.
- **Features:** 
    - Reversed list order (newest first).
    - Custom scrollbar integration.
    - Contextual action buttons (Edit ✏️ and Delete 🗑) that appear on hover.

## Styling Changes

- **File:** `app/globals.css`
- **Added:** `.custom-scrollbar` class definitions.
- **Benefit:** Cleaned up `page.tsx` and made the scrollbar style available globally for any future list components.

## How to Maintain

When adding new features:
1.  **UI Changes:** Modify the specific file in `app/components/`.
2.  **API/Logic Changes:** Update the functions (`fetchTransactions`, `tambahTransaksi`, etc.) in `app/page.tsx`.
3.  **Props:** If a component needs more data, update its `interface` in the component file and pass the new prop from `page.tsx`.
